# SQL Data Types

In SQL, data types are used to define the type of data that can be stored in a column of a table. Each column in a table is associated with a specific data type that determines the kind of values that can be stored in that column. Understanding data types is essential for designing efficient and well-structured databases.

## Common Data Types in SQL

SQL supports a wide range of data types that can be broadly categorized into the following groups:

- **Numeric Data Types**: Used to store numeric values such as integers, decimals, and floating-point numbers.
- **String Data Types**: Used to store character strings such as text and alphanumeric values.
- **Date/Time Data Types**: Used to store date and time values.
- **Special Data Types**: Used to store special values such as binary data, JSON, and XML.

Let's explore each of these data types in more detail.

### Numeric Data Types

- **INT**: Used to store whole numbers (integers) within a specific range.

```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    quantity INT -- the quantity is an integer
);

INSERT INTO products (id, quantity) VALUES (1, 10); -- Insert an integer
```

- **DECIMAL**: Used to store fixed-point numbers with a specified precision and scale.
```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    price DECIMAL(10, 2) -- the price is a decimal number with 10 digits and 2 decimal places
);

INSERT INTO products (id, price) VALUES (1, 19.99); -- Insert a decimal number
```

- **FLOAT**: Used to store floating-point numbers with a specified precision.

```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    weight FLOAT -- the weight is a floating-point number
);

INSERT INTO products (id, weight) VALUES (1, 2.5); -- Insert a floating-point number
```

- **DOUBLE**: Used to store double-precision floating-point numbers.
```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    value DOUBLE -- the value is a double-precision floating-point number
);

INSERT INTO products (id, value) VALUES (1, 123.45); -- Insert a double-precision floating-point number
```

### String Data types

- **CHAR**: Used to store fixed-length character strings.

```sql
CREATE TABLE customers (
    id INT PRIMARY KEY,
    name CHAR(50) -- the name is a fixed-length string with a maximum length of 50 characters
);

INSERT INTO customers (id, name) VALUES (1, 'John'); -- Insert a fixed-length string
```

> **Note:** whenever you define a **CHAR** data type, it will always store the fixed length of characters. If you store a string with fewer characters, it will pad the remaining characters with spaces. like if you store a string 'John' in a **CHAR(10)** column, it will be stored as 'John--------' (padded with spaces).

- **VARCHAR**: Used to store variable-length character strings.
    
```sql
CREATE TABLE customers (
    id INT PRIMARY KEY,
    email VARCHAR(100) -- the email is a variable-length string with a maximum length of 100 characters
);

INSERT INTO customers (id, email) VALUES (1, 'hello@gmail.com'); -- Insert a variable-length string
```

> **Note:** **VARCHAR** data type is used when you want to store a string with a variable length. It will only store the actual length of the string without any padding. For example, if you store a string 'John' in a **VARCHAR(10)** column, it will be stored as 'John' (without any padding).

- **TEXT**: Used to store large text values.

```sql
CREATE TABLE customers (
    id INT PRIMARY KEY,
    address TEXT -- the address is a large text value
);

INSERT INTO customers (id, address) VALUES (1, '123 Main Street, City, Country'); -- Insert a large text value
```

### Date/Time Data types

- **DATE**: Used to store date values in the format **YYYY-MM-DD**.

```sql
CREATE TABLE orders (
    id INT PRIMARY KEY,
    order_date DATE -- the order_date is a date value
);

INSERT INTO orders (id, order_date) VALUES (1, '2022-10-15'); -- Insert a date value
```

- **TIME**: Used to store time values in the format **HH:MM:SS**.

```sql
CREATE TABLE orders (
    id INT PRIMARY KEY,
    delivery_time TIME -- the delivery_time is a time value
);

INSERT INTO orders (id, delivery_time) VALUES (1, '14:30:00'); -- Insert a time value
```

- **DATETIME**: Used to store date and time values in the format **YYYY-MM-DD HH:MM:SS**.

```sql
CREATE TABLE orders (
    id INT PRIMARY KEY,
    created_at DATETIME -- the created_at is a date and time value
);

INSERT INTO orders (id, created_at) VALUES (1, '2022-10-15 14:30:00'); -- Insert a date and time value
```

### Special Data types
- **BLOB**: Used to store binary large objects such as images and multimedia files.

```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    image BLOB -- the image is a binary large object
);

INSERT INTO products (id, image) VALUES (1, 'binary data'); -- Insert binary data
```

- **JSON**: Used to store JSON (JavaScript Object Notation) data.

```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    details JSON -- the details are JSON data
);

INSERT INTO products (id, details) VALUES (1, '{"name": "Product 1", "price": 19.99}'); -- Insert JSON data
```

- **XML**: Used to store XML (eXtensible Markup Language) data.

```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    description XML -- the description is XML data
);

INSERT INTO products (id, description) VALUES (1, '<product><name>Product 1</name><price>19.99</price></product>'); -- Insert XML data
```
